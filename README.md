# 이유경 포트폴리오

개인 포트폴리오 웹페이지입니다.  
강한 타이포그래피, 중앙 비디오, 인터랙티브 프로젝트 섹션을 중심으로 구성했습니다.

## 구성

- `index.html`: 페이지 구조와 콘텐츠
- `styles.css`: 레이아웃, 반응형 스타일, 인터랙션 스타일
- `main.js`: 스크롤, 프로젝트 이미지 팝업, 모션 제어
- `assets/video/hero.mov`: 첫 화면 비디오
- `assets/`: 아이콘, 프로필 이미지, 프로젝트 캡처 이미지
- `llms.txt`: AI/LLM 검색용 사이트 요약
- `robots.txt`, `sitemap.xml`: 검색 엔진용 파일

## AX 활용 방식

- **제공한 자료**
  - `assets/video/hero.mov`
  - 자기소개 PDF
  - 직접 작성한 자기소개 문장
  - 프로젝트별 경험 설명
  - GitHub, Blog, LinkedIn 링크
  - favicon, 종이 질감 이미지 등 시각 자료

- **자료 제공 방식**
  - 파일 경로와 첨부 이미지를 직접 전달
  - PDF 내용을 소개, 경력, 프로젝트 문구에 반영하도록 요청
  - 필요한 문장은 직접 붙여넣어 기준 문장으로 제공
  - 프로젝트 캡처 이미지는 PDF에서 추출해 사용
  - 깨진 아이콘, 어색한 문장, 잘못된 약자는 화면 기준으로 피드백

- **프롬프트 작성 방식**
  - 전체 페이지를 한 번에 요청하지 않고 섹션 단위로 나누어 요청
  - 히어로, ABOUT ME, 티켓 카드, 프로젝트 탭, Contact 순서로 구체화
  - 색상, 배치, 모션, 반응형 조건을 화면 요소 기준으로 설명
  - 문구는 "AI 느낌 없이", "담백하게", "PDF에서 최대한 발췌"처럼 톤을 명확히 지정
  - 결과가 어색하면 어떤 부분이 어색한지 직접 지적하고 수정 방향을 다시 제시

- **검증 방식**
  - 브라우저에서 데스크톱과 모바일 화면을 확인
  - 이미지 경로, 링크, favicon, 아이콘 로딩 여부 확인
  - 프로젝트 이미지 팝업, 호버 효과, sticky 메뉴 동작 확인
  - 문구가 실제 경험과 맞는지 반복 검토
  - `node --check main.js`로 JavaScript 문법 확인

## 실행 방법

정적 웹페이지이므로 `index.html`을 직접 열 수 있습니다.

## 참고

이 프로젝트는 자동화 테스트 스크립트를 두지 않았습니다.  
레이아웃과 인터랙션은 브라우저에서 직접 확인하는 방식으로 검증했습니다.
